Modifica las condiciones de visibilidad de las clases Alumno y Profesor. Las condiciones nuevas son: "La clase Alumno se puede usar fuera del paquete, pero la clase Profesor sólo se puede usar dentro del paquete al que pertenece".

Dentro de la clase Alumno:
- El método inicialDatos() se puede usar fuera de la clase, pero no fuera del paquete, a menos que herede la clase Alumno.
- El método visualDatos() se puede usar desde cualquier clase (independientemente del paquete donde esté).
- El método modificarDatos() sólo se puede usar dentro de las clases del mismo paquete.
- Los métodos pedirNombre() y pedirEdad() sólo se pueden usar dentro de la propia clase Alumno.


Dentro de la clase Profesor:
- El método pedirTodosDatos() se puede usar fuera de la clase, pero sólo por las clases que están dentro de su mismo paquete.
Los métodos pedirNombre() y pedirEspecialidad() sólo se pueden usar dentro de la propia clase.


El método visualDatosProfesor() se puede usar fuera de la clase (independientemente del paquete donde esté). En principio, como no se puede usar la clase Profesor fuera del paquete, no se tendrá acceso a dicho método; no obstante, se deja bien diseñado para que cuando cambie esta restricción de la clase, automáticamente ya se pueda acceder a dicho método.
